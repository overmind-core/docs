This page covers your first few minutes in Overmind. You will create an account, set up a project, and bring in your first agent. It takes about ten minutes.

## 1. Create your account

Sign up at [console.overmindlab.ai](https://console.overmindlab.ai). You can register with an email and password, or sign in with your work account.

{/* TODO(screenshot): Sign-up / login screen */}

## 2. Tell us about your goals

The first time you sign in, Overmind asks a few short questions. It wants to know what you are building, what matters most to you (for example accuracy, cost, or speed), and what you would like to improve. Your answers shape the suggestions you see later. You can skip this and come back to it whenever you like.

{/* TODO(screenshot): Onboarding questionnaire (goals step) */}

## 3. Create a project

A project holds your agents, data, and results. Most teams start with one project per product or use case. You can create more later and switch between them from the project menu.

{/* TODO(screenshot): Create-project dialog and project switcher */}

## 4. Bring in your first agent

Once your project is ready, a setup checklist walks you through getting your first agent and some data into Overmind.

1. Connect your code so Overmind can read your agent. See [Integrations](/onboarding/integrations).
2. Bring in your traces so Overmind can see how the agent behaves in production. See [Datasets and Traces](/context/datasets-and-traces).
3. Review the agent that Overmind builds from your code and data. See [Agents](/context/agents).

{/* TODO(screenshot): Get-started checklist with steps completing */}

## 5. Create an API key

If you want to send data to Overmind from your own systems, create an API key in your project settings. The key grants access to your project, so keep it secret. You can create more than one key and revoke any of them at any time.

{/* TODO(screenshot): API keys section in project settings */}

## Good to know

The checklist always shows your next step, so you never have to guess where to go. Your data stays inside your project and your team, and you can revisit onboarding, switch projects, or rotate keys whenever you need to.
