The `client.traces` resource provides read access to the OpenTelemetry trace data that Overmind has collected for your agents. You can list root spans across all traces, retrieve the full span tree for a specific trace, and enumerate the distinct service names visible to your account.

<Note>
  To **send** traces to Overmind, use the [`@overmind-lab/trace-sdk`](/docs/tracing/js-sdk) instrumentation package. The `overmind-lab` REST client documented here is for **reading** trace data that has already been ingested.
</Note>

---

## `client.traces.list()`

**Signature:** `client.traces.list(query?, options?) -> Promise<TraceListResponse>`

**Endpoint:** `GET /api/traces/`

Returns a paginated list of traces — one row per trace, representing the root span (`parent_span_id IS NULL`) of each trace the authenticated user can see. Filter by agent, job, service name, time range, error status, span type, and more.

### Query parameters

<ParamField query="agent" type="string" format="uuid">
  Filter traces associated with this agent UUID.
</ParamField>

<ParamField query="job" type="string" format="uuid">
  Filter traces associated with this job UUID.
</ParamField>

<ParamField query="iteration" type="string" format="uuid">
  Filter traces associated with this job iteration UUID.
</ParamField>

<ParamField query="project" type="string" format="uuid">
  Filter traces belonging to this project UUID.
</ParamField>

<ParamField query="service_name" type="string">
  Filter to traces emitted by this service.
</ParamField>

<ParamField query="service_name__in" type="string[]">
  Filter to traces from multiple services. Pass multiple values as comma-separated strings.
</ParamField>

<ParamField query="span_type" type="string">
  Filter by span type: `llm_call` or `tool_call`.
</ParamField>

<ParamField query="has_error" type="boolean">
  When `true`, return only traces that contain at least one error span.
</ParamField>

<ParamField query="name" type="string">
  Filter by root span name.
</ParamField>

<ParamField query="operation" type="string">
  Filter by root span operation string.
</ParamField>

<ParamField query="trace_id" type="string">
  Filter by the OTel 32-character hex trace ID.
</ParamField>

<ParamField query="span_id" type="string">
  Filter by root span ID.
</ParamField>

<ParamField query="received_at__gte" type="string" format="date-time">
  Return traces received at or after this ISO 8601 timestamp.
</ParamField>

<ParamField query="received_at__lte" type="string" format="date-time">
  Return traces received at or before this ISO 8601 timestamp.
</ParamField>

<ParamField query="min_duration_ms" type="number">
  Return traces with duration at or above this value (milliseconds).
</ParamField>

<ParamField query="max_duration_ms" type="number">
  Return traces with duration at or below this value (milliseconds).
</ParamField>

<ParamField query="ordering" type="string">
  Field to sort by. Prefix with `-` for descending order (e.g. `-received_at`).
</ParamField>

<ParamField query="page" type="number">
  Page number within the paginated result set.
</ParamField>

<ParamField query="search" type="string">
  Full-text search term.
</ParamField>

### Example

```typescript
import OvermindLab from 'overmind-lab';

const client = new OvermindLab();

// List all traces
const traces = await client.traces.list();
console.log(traces.count);

// Filter to a specific agent and time window
const filtered = await client.traces.list({
  agent: '182bd5e5-6e1a-4fe4-a799-aa6d9a6ab26e',
  received_at__gte: '2024-01-01T00:00:00Z',
  received_at__lte: '2024-01-31T23:59:59Z',
  has_error: false,
  ordering: '-received_at',
});

for (const trace of filtered.results) {
  console.log(
    `${trace.trace_id} — ${trace.service_name} — ${trace.duration_ns}ns`,
  );
}
```

## `client.traces.retrieve()`

**Signature:** `client.traces.retrieve(traceId: string, options?) -> Promise<TraceRetrieveResponse>`

**Endpoint:** `GET /api/traces/{trace_id}/`

Fetches a single trace by its OpenTelemetry `trace_id` (the 32-character hex string). Returns the root span plus every span sharing that `trace_id`, along with a total `span_count`.

### Path parameters

<ParamField path="traceId" type="string" required>
  The OTel 32-character hex trace ID. This is the `trace_id` field returned in list results, **not** the row UUID.
</ParamField>

### Response fields

| Field         | Type              | Description                                               |
|---------------|-------------------|-----------------------------------------------------------|
| `trace_id`    | `string`          | The OTel trace ID used to look up this trace.             |
| `span_count`  | `number`          | Total number of spans in this trace.                      |
| `root`        | `Root \| null`    | The root span object, or `null` if none was found.        |
| `spans`       | `Span[]`          | All spans sharing this `trace_id`, including the root.    |

Each span in `root` and `spans` contains:

| Field             | Type                        | Description                                      |
|-------------------|-----------------------------|--------------------------------------------------|
| `trace_id`        | `string`                    | OTel trace ID.                                   |
| `span_id`         | `string`                    | Unique span ID.                                  |
| `parent_span_id`  | `string \| null`            | Parent span ID, or `null` for the root span.     |
| `name`            | `string`                    | Span operation name.                             |
| `operation`       | `string`                    | High-level operation category.                   |
| `service_name`    | `string`                    | Originating service name.                        |
| `span_type`       | `"llm_call" \| "tool_call"` | Semantic span type.                              |
| `start_time_ns`   | `number`                    | Start time as Unix epoch nanoseconds.            |
| `end_time_ns`     | `number`                    | End time as Unix epoch nanoseconds.              |
| `duration_ns`     | `number`                    | Duration in nanoseconds.                         |
| `status_code`     | `number`                    | OTel status code.                                |
| `status_message`  | `string`                    | Human-readable status message.                   |
| `is_root`         | `boolean`                   | Whether this is the root span for the trace.     |
| `attributes`      | `unknown`                   | OTel span attributes.                            |
| `events`          | `unknown`                   | Span events.                                     |
| `links`           | `unknown`                   | Span links.                                      |
| `resource_attrs`  | `unknown`                   | Resource-level OTel attributes.                  |
| `agent`           | `string \| null` (uuid)     | Associated Overmind agent UUID.                  |
| `job`             | `string \| null` (uuid)     | Associated Overmind job UUID.                    |
| `iteration`       | `string \| null` (uuid)     | Associated job iteration UUID.                   |
| `project`         | `string` (uuid)             | Owning project UUID.                             |
| `received_at`     | `string` (date-time)        | When Overmind received this span.                |

### Example

```typescript
import OvermindLab from 'overmind-lab';

const client = new OvermindLab();

const trace = await client.traces.retrieve('4bf92f3577b34da6a3ce929d0e0e4736');

console.log(`Trace ID: ${trace.trace_id}`);
console.log(`Total spans: ${trace.span_count}`);

if (trace.root) {
  console.log(`Root span: ${trace.root.name} (${trace.root.duration_ns}ns)`);
}

for (const span of trace.spans) {
  console.log(`  [${span.span_type}] ${span.name} — ${span.service_name}`);
}
```

## `client.traces.listServices()`

**Signature:** `client.traces.listServices(query?, options?) -> Promise<TraceListServicesResponse>`

**Endpoint:** `GET /api/traces/services/`

Returns a paginated list of distinct service names visible to the authenticated user. Use this to populate filter dropdowns or to audit which services are emitting trace data to Overmind.

The response `results` array contains plain strings (service name values), not objects. The same filter parameters from `client.traces.list()` are accepted, so you can scope the service list to a specific agent, job, or time window.

### Query parameters

<ParamField query="agent" type="string" format="uuid">
  Scope to service names seen in traces for this agent UUID.
</ParamField>

<ParamField query="job" type="string" format="uuid">
  Scope to service names seen in traces for this job UUID.
</ParamField>

<ParamField query="project" type="string" format="uuid">
  Scope to service names seen in traces for this project UUID.
</ParamField>

<ParamField query="received_at__gte" type="string" format="date-time">
  Include only service names from traces received at or after this timestamp.
</ParamField>

<ParamField query="received_at__lte" type="string" format="date-time">
  Include only service names from traces received at or before this timestamp.
</ParamField>

<ParamField query="has_error" type="boolean">
  When `true`, return only service names that appear in error traces.
</ParamField>

<ParamField query="page" type="number">
  Page number within the paginated result set.
</ParamField>

<ParamField query="ordering" type="string">
  Field to sort by. Prefix with `-` for descending order.
</ParamField>

### Example

```typescript
import OvermindLab from 'overmind-lab';

const client = new OvermindLab();

// List all service names
const services = await client.traces.listServices();
console.log(`${services.count} services found:`);
for (const name of services.results) {
  console.log(' -', name);
}

// Scope to a specific agent
const agentServices = await client.traces.listServices({
  agent: '182bd5e5-6e1a-4fe4-a799-aa6d9a6ab26e',
});
console.log(agentServices.results);
```
