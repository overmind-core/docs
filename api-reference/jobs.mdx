A **Job** is an optimization run tied to a specific agent. When you launch a job, Overmind runs your agent against a dataset, scores the outputs, and — depending on the job type — proposes and evaluates candidate improvements across multiple iterations. Each job tracks its progress through a `status` field and records every candidate it tested as a nested **Job Iteration**. You use jobs to understand how your agent is performing today and to drive it toward higher scores.

---

## Job types

The `job_type` field controls what Overmind does during the run:

| Value | Description |
|---|---|
| `inference` | Runs the agent on the active dataset and records raw outputs. Use this to collect a baseline before optimizing. |
| `uploaded_outputs_setup` | Configures the agent from a set of pre-collected agent outputs you upload, rather than running live inference. |
| `judge_scoring` | Runs an LLM-as-Judge scoring pass against previously collected outputs, populating scores without re-running the agent. |
| `prompt_tuning` | Iteratively generates and evaluates candidate prompt modifications, searching for versions that score higher. |
| `model_backtesting` | Runs the agent against multiple model configurations to compare performance across models. |

---

## Job status

The `status` field reflects the current state of a job:

| Value | Description |
|---|---|
| `pending` | The job has been created but has not started running yet. |
| `running` | The job is actively executing iterations. |
| `completed` | All iterations have finished and results are available. |
| `failed` | The job encountered an unrecoverable error. |
| `cancelled` | The job was manually cancelled before completion. |

## GET /api/jobs/

List all jobs visible to the authenticated user. You can filter by agent, project, status, job type, and date range.

### Query parameters

<ParamField query="agent" type="string (uuid)">
  Filter jobs belonging to a specific agent.
</ParamField>

<ParamField query="job_type" type="string">
  Filter by job type. One of: `inference`, `uploaded_outputs_setup`, `judge_scoring`, `prompt_tuning`, `model_backtesting`.
</ParamField>

<ParamField query="status" type="string">
  Filter by job status. One of: `pending`, `running`, `completed`, `failed`, `cancelled`.
</ParamField>

<ParamField query="project" type="string (uuid)">
  Filter jobs scoped to a specific project.
</ParamField>

<ParamField query="created_after" type="string (date-time)">
  Return only jobs created after this ISO 8601 timestamp.
</ParamField>

<ParamField query="created_before" type="string (date-time)">
  Return only jobs created before this ISO 8601 timestamp.
</ParamField>

<ParamField query="score_min" type="number">
  Filter to jobs whose best score is at or above this value.
</ParamField>

<ParamField query="score_max" type="number">
  Filter to jobs whose best score is at or below this value.
</ParamField>

<ParamField query="ordering" type="string">
  Field to sort results by. Prefix with `-` for descending (e.g. `-created_at`).
</ParamField>

<ParamField query="page" type="number">
  Page number within the paginated result set.
</ParamField>

### Response fields

<ResponseField name="count" type="number" required>
  Total number of jobs matching the query.
</ResponseField>

<ResponseField name="results" type="array">
  Paginated list of job summary objects.

  <Expandable title="Job summary object">
    <ResponseField name="id" type="string (uuid)" required>
      Unique job identifier.
    </ResponseField>
    <ResponseField name="created_at" type="string (date-time)" required>
      Timestamp when the job was created.
    </ResponseField>
    <ResponseField name="updated_at" type="string (date-time)" required>
      Timestamp of the most recent update.
    </ResponseField>
    <ResponseField name="agent" type="string (uuid)">
      UUID of the agent this job targets.
    </ResponseField>
    <ResponseField name="job_type" type="string">
      Type of optimization run.
    </ResponseField>
    <ResponseField name="status" type="string">
      Current job status.
    </ResponseField>
    <ResponseField name="num_iterations" type="number">
      Number of iterations requested for this job.
    </ResponseField>
    <ResponseField name="baseline_score" type="number">
      Score recorded before optimization began.
    </ResponseField>
    <ResponseField name="best_score" type="number">
      Highest score achieved across all iterations.
    </ResponseField>
    <ResponseField name="improvement" type="number">
      Score improvement over the baseline (`best_score - baseline_score`).
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="next" type="string (uri)">
  URL of the next page, or `null`.
</ResponseField>

<ResponseField name="previous" type="string (uri)">
  URL of the previous page, or `null`.
</ResponseField>

### Example

```bash
curl 'https://api.overmindlab.ai/api/jobs/?status=completed&job_type=prompt_tuning' \
  -H 'Authorization: Bearer $OVERMIND_API_KEY'
```

## POST /api/jobs/

Create and enqueue a new optimization job. The job starts asynchronously — poll `GET /api/jobs/{id}/` or check the Overmind dashboard to track its progress.

### Body parameters

<ParamField body="agent" type="string (uuid)">
  UUID of the agent to run this job against.
</ParamField>

<ParamField body="job_type" type="string">
  Type of job to run. One of: `inference`, `uploaded_outputs_setup`, `judge_scoring`, `prompt_tuning`, `model_backtesting`.
</ParamField>

<ParamField body="num_iterations" type="number">
  Number of optimization iterations to attempt.
</ParamField>

<ParamField body="candidates_per_iteration" type="number">
  Number of candidate prompt variants to evaluate per iteration.
</ParamField>

<ParamField body="data_source" type="string">
  Source of the evaluation data. Max 20 characters.
</ParamField>

<ParamField body="analyzer_model" type="string">
  Model to use for the LLM judge during this job. Overrides the agent-level setting. Max 128 characters.
</ParamField>

<ParamField body="status" type="string">
  Initial status for the job. Defaults to `pending`.
</ParamField>

### Response fields

<ResponseField name="id" type="string (uuid)" required>
  Unique identifier for the created job.
</ResponseField>

<ResponseField name="created_at" type="string (date-time)" required>
  Timestamp when the job was created.
</ResponseField>

<ResponseField name="iterations" type="array" required>
  Array of job iteration objects (empty on creation, populated as the job runs).

  <Expandable title="Iteration object">
    <ResponseField name="id" type="string (uuid)" required>
      Unique iteration identifier.
    </ResponseField>
    <ResponseField name="iteration_name" type="string" required>
      Short human-readable name for this iteration.
    </ResponseField>
    <ResponseField name="job" type="string (uuid)" required>
      UUID of the parent job.
    </ResponseField>
    <ResponseField name="avg_score" type="number">
      Average score across all datapoints in this iteration.
    </ResponseField>
    <ResponseField name="description" type="string">
      Description of what changed in this iteration.
    </ResponseField>
    <ResponseField name="status" type="string">
      Whether this iteration was kept (`keep`) or discarded (`discard`).
    </ResponseField>
    <ResponseField name="dimension_scores" type="object">
      Per-dimension breakdown of the scores for this iteration.
    </ResponseField>
    <ResponseField name="order" type="number">
      Ordinal position of this iteration within the job.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="job_type" type="string">
  Type of job as created.
</ResponseField>

<ResponseField name="status" type="string">
  Current job status.
</ResponseField>

<ResponseField name="baseline_score" type="number">
  Score recorded before optimization.
</ResponseField>

<ResponseField name="best_score" type="number">
  Best score achieved so far.
</ResponseField>

<ResponseField name="improvement" type="number">
  Score delta over the baseline.
</ResponseField>

### Example

```bash
curl https://api.overmindlab.ai/api/jobs/ \
  -X POST \
  -H 'Authorization: Bearer $OVERMIND_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{
    "agent": "182bd5e5-6e1a-4fe4-a799-aa6d9a6ab26e",
    "job_type": "prompt_tuning",
    "num_iterations": 5,
    "candidates_per_iteration": 3
  }'
```

## GET /api/jobs/\{id\}/

Retrieve a single job with its full iteration history. Use this endpoint to poll job progress or inspect final results after a run completes.

### Path parameters

<ParamField path="id" type="string (uuid)" required>
  UUID of the job to retrieve.
</ParamField>

### Response fields

Returns the full job object as described in the `POST /api/jobs/` response above, with the `iterations` array populated with all iterations completed so far.

### Example

```bash
curl https://api.overmindlab.ai/api/jobs/182bd5e5-6e1a-4fe4-a799-aa6d9a6ab26e/ \
  -H 'Authorization: Bearer $OVERMIND_API_KEY'
```

## PATCH /api/jobs/\{id\}/

Partially update a job record. You can use this to update metadata fields or change the status of a running job.

### Path parameters

<ParamField path="id" type="string (uuid)" required>
  UUID of the job to update.
</ParamField>

### Example

```bash
curl https://api.overmindlab.ai/api/jobs/182bd5e5-6e1a-4fe4-a799-aa6d9a6ab26e/ \
  -X PATCH \
  -H 'Authorization: Bearer $OVERMIND_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{ "status": "cancelled" }'
```

## DELETE /api/jobs/\{id\}/

Delete a job and all of its associated iterations.

### Path parameters

<ParamField path="id" type="string (uuid)" required>
  UUID of the job to delete.
</ParamField>

### Example

```bash
curl https://api.overmindlab.ai/api/jobs/182bd5e5-6e1a-4fe4-a799-aa6d9a6ab26e/ \
  -X DELETE \
  -H 'Authorization: Bearer $OVERMIND_API_KEY'
```

Returns `204 No Content` on success.

## Job Iterations

Job Iterations are the individual candidate runs within a job. Overmind creates them automatically as a job progresses, but you can also query and manage them directly through the `/api/job-iterations/` endpoints.

---

### GET /api/job-iterations/

List job iterations across all jobs you have access to. Filter by job, status, or score range.

<ParamField query="job" type="string (uuid)">
  Filter iterations belonging to a specific job.
</ParamField>

<ParamField query="status" type="string">
  Filter by iteration status: `keep` or `discard`.
</ParamField>

<ParamField query="score_min" type="number">
  Return only iterations with `avg_score` at or above this value.
</ParamField>

<ParamField query="score_max" type="number">
  Return only iterations with `avg_score` at or below this value.
</ParamField>

<ParamField query="page" type="number">
  Page number within the paginated result set.
</ParamField>

```bash
curl 'https://api.overmindlab.ai/api/job-iterations/?status=keep' \
  -H 'Authorization: Bearer $OVERMIND_API_KEY'
```

---

### POST /api/job-iterations/

Create a job iteration manually. In normal usage Overmind creates iterations automatically, but you can use this endpoint to record a custom iteration.

<ParamField body="iteration_name" type="string" required>
  Short name for the iteration. Between 1 and 64 characters.
</ParamField>

<ParamField body="job" type="string (uuid)" required>
  UUID of the parent job.
</ParamField>

<ParamField body="avg_score" type="number">
  Average score for this iteration.
</ParamField>

<ParamField body="description" type="string">
  Description of what changed in this iteration.
</ParamField>

<ParamField body="status" type="string">
  `keep` or `discard`.
</ParamField>

<ParamField body="dimension_scores" type="object">
  Per-dimension score breakdown.
</ParamField>

```bash
curl https://api.overmindlab.ai/api/job-iterations/ \
  -H 'Authorization: Bearer $OVERMIND_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{
    "iteration_name": "manual-baseline",
    "job": "182bd5e5-6e1a-4fe4-a799-aa6d9a6ab26e",
    "avg_score": 62.5,
    "status": "keep"
  }'
```

---

### GET /api/job-iterations/\{id\}/

Retrieve a single job iteration by UUID.

<ParamField path="id" type="string (uuid)" required>
  UUID of the iteration to retrieve.
</ParamField>

```bash
curl https://api.overmindlab.ai/api/job-iterations/aabbccdd-1234-5678-abcd-ef0123456789/ \
  -H 'Authorization: Bearer $OVERMIND_API_KEY'
```

---

### PATCH /api/job-iterations/\{id\}/

Partially update a job iteration. The most common use is to change the `status` from `discard` to `keep` (or vice versa) to mark which candidates you want to promote.

<ParamField path="id" type="string (uuid)" required>
  UUID of the iteration to update.
</ParamField>

```bash
curl https://api.overmindlab.ai/api/job-iterations/aabbccdd-1234-5678-abcd-ef0123456789/ \
  -X PATCH \
  -H 'Authorization: Bearer $OVERMIND_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{ "status": "keep" }'
```

---

Delete a job iteration permanently.

<ParamField path="id" type="string (uuid)" required>

```bash
  -X DELETE \
  -H 'Authorization: Bearer $OVERMIND_API_KEY'
```
