**Traces** are OpenTelemetry-based recordings of every LLM call, tool invocation, and operation that your agent performs. Overmind collects these automatically when you instrument your application with the Overmind tracing SDK. Each trace represents a complete end-to-end agent invocation, composed of one or more **spans** — individual units of work such as a single LLM call or a tool call. Spans carry rich attributes including model name, prompt and completion text, token counts, latency in nanoseconds, status codes, and links to the job or iteration that produced them. You can use the Traces API to query across your recorded runs, drill into individual trace trees, and discover which services are sending data.

<Note>
  Traces are created automatically by the Overmind tracing SDKs when you instrument your application — you don't create them via the API directly. See the [tracing guide](/docs/tracing/overview) for instructions on how to instrument your agent.
</Note>

## Trace data model

A **trace** is identified by a `trace_id` — a 32-character hexadecimal string following the OpenTelemetry standard. Every trace has exactly one root span (the span with no `parent_span_id`) and zero or more child spans that share the same `trace_id`.

A **span** contains:

| Field | Description |
|---|---|
| `trace_id` | 32-char hex trace identifier shared by all spans in the trace. |
| `span_id` | Unique identifier for this individual span. |
| `parent_span_id` | ID of the parent span. `null` on the root span. |
| `name` | Operation name (e.g. `ChatCompletion`, `search_tool`). |
| `span_type` | Either `llm_call` or `tool_call`. |
| `service_name` | Name of the service that emitted this span. |
| `start_time_ns` | Epoch start time in nanoseconds. |
| `end_time_ns` | Epoch end time in nanoseconds. |
| `duration_ns` | Elapsed time in nanoseconds (`end_time_ns - start_time_ns`). |
| `status_code` | OpenTelemetry status code (0 = unset, 1 = OK, 2 = error). |
| `status_message` | Human-readable status message. |
| `attributes` | Key/value span attributes (e.g. model name, token counts, prompt text). |
| `events` | Timestamped events recorded during the span. |
| `is_root` | `true` if this is the root span of the trace. |
| `agent` | UUID of the Overmind agent that produced this span, if linked. |
| `job` | UUID of the optimization job this span belongs to, if any. |
| `iteration` | UUID of the job iteration this span belongs to, if any. |
| `project` | UUID of the Overmind project. |

## GET /api/traces/

List root spans — one entry per trace. This is the primary way to browse your recorded traces. Each result represents the root span of a distinct trace, giving you the top-level view of every agent invocation. Filter by service name, agent, time range, duration, and more.

### Query parameters

<ParamField query="service_name" type="string">
  Filter traces emitted by a specific service name.
</ParamField>

<ParamField query="agent" type="string (uuid)">
  Filter traces linked to a specific agent.
</ParamField>

<ParamField query="job" type="string (uuid)">
  Filter traces belonging to a specific job run.
</ParamField>

<ParamField query="iteration" type="string (uuid)">
  Filter traces belonging to a specific job iteration.
</ParamField>

<ParamField query="project" type="string (uuid)">
  Filter traces scoped to a specific project.
</ParamField>

<ParamField query="span_type" type="string">
  Filter by span type. One of: `llm_call`, `tool_call`.
</ParamField>

<ParamField query="has_error" type="boolean">
  When `true`, return only traces that contain at least one errored span.
</ParamField>

<ParamField query="min_duration_ms" type="number">
  Return only traces with total duration at or above this value (milliseconds).
</ParamField>

<ParamField query="max_duration_ms" type="number">
  Return only traces with total duration at or below this value (milliseconds).
</ParamField>

<ParamField query="received_at__gte" type="string (date-time)">
  Return only traces received at or after this ISO 8601 timestamp.
</ParamField>

<ParamField query="received_at__lte" type="string (date-time)">
  Return only traces received at or before this ISO 8601 timestamp.
</ParamField>

<ParamField query="name" type="string">
  Filter by root span name.
</ParamField>

<ParamField query="trace_id" type="string">
  Look up a specific trace by its OTel trace ID string.
</ParamField>

<ParamField query="ordering" type="string">
  Field to sort results by. Prefix with `-` for descending (e.g. `-received_at`).
</ParamField>

<ParamField query="page" type="number">
  Page number within the paginated result set.
</ParamField>

### Response fields

<ResponseField name="count" type="number" required>
  Total number of traces matching the query.
</ResponseField>

<ResponseField name="results" type="array">
  Paginated list of root span objects.

  <Expandable title="Root span object">
    <ResponseField name="trace_id" type="string" required>
      32-character hex trace identifier.
    </ResponseField>
    <ResponseField name="span_id" type="string" required>
      Identifier for this root span.
    </ResponseField>
    <ResponseField name="name" type="string" required>
      Name of the root operation.
    </ResponseField>
    <ResponseField name="service_name" type="string" required>
      Service that emitted this trace.
    </ResponseField>
    <ResponseField name="span_type" type="string" required>
      `llm_call` or `tool_call`.
    </ResponseField>
    <ResponseField name="start_time_ns" type="number" required>
      Epoch start time in nanoseconds.
    </ResponseField>
    <ResponseField name="end_time_ns" type="number" required>
      Epoch end time in nanoseconds.
    </ResponseField>
    <ResponseField name="duration_ns" type="number" required>
      Total trace duration in nanoseconds.
    </ResponseField>
    <ResponseField name="status_code" type="number" required>
      OTel status code.
    </ResponseField>
    <ResponseField name="status_message" type="string" required>
      Human-readable status.
    </ResponseField>
    <ResponseField name="received_at" type="string (date-time)" required>
      When Overmind received this span.
    </ResponseField>
    <ResponseField name="agent" type="string (uuid)" required>
      Linked agent UUID.
    </ResponseField>
    <ResponseField name="job" type="string (uuid)" required>
      Linked job UUID.
    </ResponseField>
    <ResponseField name="project" type="string (uuid)" required>
      Linked project UUID.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="next" type="string (uri)">
  URL of the next page, or `null`.
</ResponseField>

<ResponseField name="previous" type="string (uri)">
  URL of the previous page, or `null`.
</ResponseField>

### Example

```bash
curl 'https://api.overmindlab.ai/api/traces/?service_name=support-agent&has_error=false' \
  -H 'Authorization: Bearer $OVERMIND_API_KEY'
```

## GET /api/traces/\{trace\_id\}/

Retrieve a complete trace tree by its OTel `trace_id`. The response includes the root span and every child span that shares the same `trace_id`. You can use this to reconstruct the full call tree for any agent invocation. Note that the path parameter is the OpenTelemetry 32-character hex trace ID, not a UUID.

### Path parameters

<ParamField path="trace_id" type="string" required>
  The OpenTelemetry 32-character hexadecimal trace ID (e.g. `4bf92f3577b34da6a3ce929d0e0e4736`).
</ParamField>

### Response fields

<ResponseField name="trace_id" type="string" required>
  The trace ID you requested.
</ResponseField>

<ResponseField name="span_count" type="number" required>
  Total number of spans in this trace.
</ResponseField>

<ResponseField name="root" type="object" required>
  The root span (the span where `parent_span_id` is null). Contains all span fields listed in the data model above plus `attributes`, `events`, `links`, `resource_attrs`, `scope_name`, and `scope_version`.
</ResponseField>

<ResponseField name="spans" type="array" required>
  All spans in the trace (including the root). Each span contains the full set of span fields.

  <Expandable title="Full span object">
    <ResponseField name="trace_id" type="string" required>Trace identifier.</ResponseField>
    <ResponseField name="span_id" type="string" required>Unique span identifier.</ResponseField>
    <ResponseField name="parent_span_id" type="string" required>Parent span ID, or empty string for the root.</ResponseField>
    <ResponseField name="name" type="string" required>Operation name.</ResponseField>
    <ResponseField name="span_type" type="string" required>`llm_call` or `tool_call`.</ResponseField>
    <ResponseField name="service_name" type="string" required>Emitting service name.</ResponseField>
    <ResponseField name="start_time_ns" type="number" required>Epoch start time in nanoseconds.</ResponseField>
    <ResponseField name="end_time_ns" type="number" required>Epoch end time in nanoseconds.</ResponseField>
    <ResponseField name="duration_ns" type="number" required>Duration in nanoseconds.</ResponseField>
    <ResponseField name="status_code" type="number" required>OTel status code.</ResponseField>
    <ResponseField name="status_message" type="string" required>Status message.</ResponseField>
    <ResponseField name="attributes" type="object" required>Key/value span attributes (model, tokens, prompt text, etc.).</ResponseField>
    <ResponseField name="events" type="object" required>Timestamped events recorded during the span.</ResponseField>
    <ResponseField name="is_root" type="boolean" required>`true` for the root span.</ResponseField>
    <ResponseField name="agent" type="string (uuid)" required>Linked agent UUID.</ResponseField>
    <ResponseField name="job" type="string (uuid)" required>Linked job UUID.</ResponseField>
    <ResponseField name="iteration" type="string (uuid)" required>Linked iteration UUID.</ResponseField>
    <ResponseField name="project" type="string (uuid)" required>Linked project UUID.</ResponseField>
  </Expandable>
</ResponseField>

curl https://api.overmindlab.ai/api/traces/4bf92f3577b34da6a3ce929d0e0e4736/ \
  -H 'Authorization: Bearer $OVERMIND_API_KEY'

## GET /api/traces/services/

List all distinct service names that have sent traces to Overmind and are visible to the authenticated user. Use this endpoint to discover which services are instrumented and to populate service filter dropdowns in your own tooling.

### Query parameters

<ParamField query="project" type="string (uuid)">
  Filter services to those that have sent traces for a specific project.
</ParamField>

<ParamField query="agent" type="string (uuid)">
  Filter services to those linked to a specific agent.
</ParamField>

<ParamField query="page" type="number">
  Page number within the paginated result set.
</ParamField>

### Response fields

<ResponseField name="count" type="number" required>
  Total number of distinct service names.
</ResponseField>

<ResponseField name="results" type="array of string" required>
  Paginated list of service name strings.
</ResponseField>

<ResponseField name="next" type="string (uri)">
  URL of the next page, or `null`.
</ResponseField>

<ResponseField name="previous" type="string (uri)">
  URL of the previous page, or `null`.
</ResponseField>

### Example

```bash
curl https://api.overmindlab.ai/api/traces/services/ \
  -H 'Authorization: Bearer $OVERMIND_API_KEY'
```

```json
{
  "count": 3,
  "results": [
    "support-agent",
    "sales-copilot",
    "document-qa"
  ],
  "next": null,
  "previous": null
}
```
