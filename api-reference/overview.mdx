The Overmind REST API gives you programmatic access to every resource on the platform — agents, optimization jobs, datasets, traces, and projects. All requests go to the base URL `https://api.overmindlab.ai`. Every endpoint returns JSON and requires an API key for authentication.

## Client libraries

Overmind publishes official client libraries for Python and TypeScript. Install the stable release from the package registry, or pin to the staging build to test pre-release features.

## Authentication

All API requests must include a valid API key in the `Authorization` header:

```
Authorization: Bearer <your-api-key>
```

See the [Authentication page](/api-reference/authentication) for full details on creating, listing, and revoking API keys.

## All endpoints

### Agents

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/agents/` | List agents |
| `POST` | `/api/agents/` | Create agent |
| `GET` | `/api/agents/{id}/` | Get agent |
| `PATCH` | `/api/agents/{id}/` | Update agent |
| `DELETE` | `/api/agents/{id}/` | Soft-delete agent |
| `GET` | `/api/agents/{id}/eval_spec/` | Get agent eval spec |

### Auth

## Request format

Send all request bodies as JSON and include a `Content-Type: application/json` header. Path parameters are written in `{id}` format in this reference — substitute the actual UUID value when making requests. Use `PATCH` for partial updates; you only need to include the fields you want to change.

## Pagination

List endpoints return a paginated envelope:

| Field | Type | Description |
|-------|------|-------------|
| `count` | number | Total number of results across all pages |
| `results` | array | Items on the current page |
| `next` | string \| null | URL of the next page, or `null` if you are on the last page |
| `previous` | string \| null | URL of the previous page, or `null` if you are on the first page |

Use the `page` query parameter to request a specific page number:

```bash
curl "https://api.overmindlab.ai/api/agents/?page=2" \
  -H 'Authorization: Bearer ovr_your_key_here'
```

## Error codes

| Status | Meaning |
|--------|---------|
| `400 Bad Request` | The request body or query parameters are invalid. Check the response body for field-level error details. |
| `401 Unauthorized` | No API key was provided, or the key is invalid or expired. |
| `403 Forbidden` | Your API key is valid but does not have permission to perform this action. |
| `404 Not Found` | The requested resource does not exist, or you do not have access to it. |
| `429 Too Many Requests` | You have exceeded the rate limit. Back off and retry after the period indicated in the `Retry-After` header. |
| `500 Internal Server Error` | An unexpected error occurred on the Overmind servers. |
