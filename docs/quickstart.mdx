In about 10 minutes you'll install Overmind, initialize it in your agent's project, register your agent, generate a policy spec and test dataset, and kick off your first autonomous optimization run. The entire workflow comes down to four terminal commands and three IDE slash commands — Overmind reads your codebase, asks only what it can't infer, and handles the rest.

<Info>
  **Requirements before you begin:**

  - Python 3.10 or higher
  - Cursor or Claude Code (for the Agent Skills slash commands)
  - API keys for at least one LLM provider — OpenAI, Anthropic, or both
</Info>

## Run your first optimization

<Steps>
  <Step title="Install Overmind">
    Install the Overmind package from PyPI:

    ```bash
    pip install overmind
    ```
  </Step>

  <Step title="Initialize your project">
    Navigate to your agent's project root and run `overmind init`:

    ```bash
    cd your-agent-project
    overmind init
    ```

    This command creates a `.overmind/` configuration directory, collects your API keys (Overmind, OpenAI, Anthropic), sets your analyzer model, and installs the Overmind Agent Skills into your IDE. You can re-run it at any time to update your configuration.
  </Step>

  <Step title="Register your agent">
    Open your project in Cursor or Claude Code. In the AI chat panel, run the register skill with the path to your agent's entrypoint file:

    ```text
    /overmind-register-agent path/to/your/agent.py
    ```

    The skill reads your codebase, identifies your agent's input/output contract and entrypoint function, and registers the agent with the Overmind backend. It will ask clarifying questions about anything it can't infer from the code.
  </Step>

  <Step title="Generate a policy, spec, and dataset">
    Still in the chat panel, run the spec and dataset generation skill:

    ```text
    /overmind-generate-spec-and-dataset my-agent
    ```

    This skill analyzes your agent code and any existing policy documents you provide, infers or refines a policy describing correct agent behavior, generates a machine-readable evaluation spec, and synthesizes a test dataset. You'll review and approve the policy in a conversational loop before it's finalized and pushed to the backend.
  </Step>

  <Step title="Run optimization">
    Kick off your first autonomous optimization run:

    ```text
    /overmind-optimize-agent my-agent
    ```

    Overmind will execute the full optimization loop — running your agent on the test dataset, scoring outputs, diagnosing failures, generating candidate code fixes, and accepting only the changes that measurably improve your agent. This may run for several iterations depending on your configuration.
  </Step>

  <Step title="View your results">
    Navigate to [console.overmindlab.ai/agents](https://console.overmindlab.ai/agents) and select your agent. You'll find the full score history across every iteration, detailed traces of each agent run, the best-performing agent version, and a diff report summarizing what changed and why.
  </Step>
</Steps>

## Instrument a running app (optional)

If you want Overmind traces from a deployed application — independently of the optimization workflow — install the Python or JS/TS tracing SDK and call the init function once at startup.

### Python

Install the SDK alongside your LLM provider:

```bash
pip install overmind
```

Then call `overmind.init()` at the top of your entry point, before any LLM calls or framework setup:

```python
import overmind

overmind.init(service_name="my-service", environment="production")
```

After that single call, every OpenAI, Anthropic, or Gemini request is automatically captured with model, inputs/outputs, latency, token counts, and cost. See the [Python SDK reference](/docs/tracing/python-sdk) for the full parameter table and manual instrumentation helpers.

### JavaScript / TypeScript

Install the SDK and your LLM provider:

```bash
npm install @overmind-lab/trace-sdk openai
```

Then initialize tracing once at startup:

```typescript
import { OpenAI } from "openai";
import { OvermindClient } from "@overmind-lab/trace-sdk";

const overmindClient = new OvermindClient({
  apiKey: process.env.OVERMIND_API_KEY!,
  appName: "my app",
});

overmindClient.initTracing({ enabledProviders: { openai: OpenAI } });
```

After `initTracing()`, all OpenAI calls in your application are traced automatically. See the [JS/TS SDK reference](/docs/tracing/js-sdk) for providers, batching, environment variables, and shutdown patterns.

<Tip>
  Tracing is independent of optimization. You can use one without the other — but continuous production traces give the optimizer more signal and lead to better results over time.
</Tip>
